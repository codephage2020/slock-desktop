# Slock Desktop Design Context

## Design Context

### Users
Slock Desktop is for people using Slock as a daily AI collaboration workspace. They need a stable desktop shell that opens the original `app.slock.ai` page, keeps local service and update controls available, and lets them tune the workspace appearance without leaving flow.

### Brand Personality
Calm, precise, collaborative. The interface should feel like a professional workbench for long sessions: quiet surfaces, clear hierarchy, and restrained status color.

### Aesthetic Direction
Use a ChatGPT-style light workspace as the primary reference: soft neutral canvas, subtle panel separation, rounded row controls, and low-distraction hover states. Use VS Code-style dark hierarchy for dark themes: layered graphite surfaces, muted text, and compact operational clarity.

### Design Principles
- One surface language across startup console, workspace injection, desktop settings, service controls, update states, dialogs, empty states, and mobile layouts.
- Theme tokens are the source of truth: `canvas`, `surface`, `surfaceStrong`, `line`, `text`, `muted`, `accent`, and `accentSoft`.
- Radius scale is fixed at `10 / 14 / 18 / 24 / pill`; new UI should map to this scale.
- Accent color is for focus, status, and selected states; large high-saturation blocks are avoided.
- Interactions are low-noise: light hover lift, `scale(0.96)` press, visible focus, and reduced-motion support.
