# Slock Desktop Design Context

## Design Context

### Users
Slock Desktop is for people using Slock as a daily AI collaboration workspace. They need a stable desktop shell that opens the original `app.slock.ai` page, keeps local service and update controls available, and lets them tune the workspace appearance without leaving flow.

### Brand Personality
Calm, precise, collaborative. The interface should feel like a professional workbench for long sessions: quiet surfaces, clear hierarchy, and restrained status color.

### Aesthetic Direction
Use a refined macOS-first productivity shell as the primary reference: calm neutral light surfaces, subtle panel separation, typography-led structure, and low-distraction hover states. The product should feel native, durable, and comfortable for long daily use.

### Design Principles
- One surface language across startup console, workspace injection, desktop settings, service controls, update states, dialogs, empty states, and mobile layouts.
- Theme tokens are the source of truth: `appBg`, `toolbarBg`, `sidebarBg`, `panelBg`, `surface`, `surfaceSecondary`, `surfaceTertiary`, `border`, `borderStrong`, `textPrimary`, `textSecondary`, `textTertiary`, `accent`, `accentHover`, `accentActive`, `selection`, `hover`, `pressed`, and `focusRing`.
- Theme is a color scheme choice. Mode is a separate light/dark/system choice that determines the surface palette for every theme.
- Custom theme support lets users define a saved accent theme while preserving the same light/dark/system surface model.
- Language is a separate `zh-CN / en-US / system` setting shared by the startup page, Desktop Settings overlay, workspace settings entry, and Tauri native title/menu text where available.
- Radius scale is fixed at `8 / 10 / 12 / 16 / 20 / pill`; new UI should map to this scale.
- Accent color is for primary action, focus, status, and selected states; large high-saturation blocks are avoided.
- Interactions are low-noise: subtle hover background, `scale(0.97)` press, visible focus, and reduced-motion support.
- Layout uses desktop-first panels: 64px toolbar, 280px sidebar, 340px contextual panel, 56px topbar, 24px page padding, and 840px readable content width.

### Component Rules
- App shell uses neutral background tone steps and thin borders for hierarchy.
- Toolbar is narrow and icon-first; sidebar is supportive with stable row rhythm.
- Main content is visually dominant and uses editorial spacing before cards.
- Right panel is optional/contextual and uses panel tone plus a left border.
- Buttons use one primary action per view; secondary and icon buttons stay neutral.
- Messages read as content blocks. Assistant content stays transparent; user content can use a subtle secondary surface.
- The startup page, nested settings page, Tauri native window theme, Desktop Settings overlay, and Launch-loaded `app.slock.ai` page must share the same active theme and mode.
- Theme color, appearance mode, language, and custom theme values are persisted as one settings model and should be validated together.
